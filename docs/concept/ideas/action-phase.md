
# ローグライクゲームにおける行動フェーズ・成否判定の基本設計

Gemini 生成

キャラクター（プレイヤーおよびエネミー）が1つの行動（移動、攻撃、アイテム使用、スキル発動など）を決定してから、その効果が世界に反映され、次のターンへ移行するまでのサイクルを5つの段階に分離して定義します。

---

## 1. 全体フロー図

```text
[1. 企図 (Intent)] ➔ [2. 予備（Check/Pre-Action）] ➔ [3. 実行 (Execution)]
⬇
[5. 事後 (Resolution)] ⬅ [4. 成否結果 (Outcome/Result)] 支流（Success/Failure）
```

---

## 2. 各段階（フェーズ）の詳細定義

### 第1段階：企図（きと）

- **日本語:** 企図 / 行動宣言 / 入力受付

- **英語:** **Intent** (インテント) / **Declaration** (デクラレーション)
- **概要:** キャラクターが「この行動をしたい」と決定した（またはプレイヤーがキー入力をした）状態。
- **ローグライクにおける役割:** * ターゲット（方向や座標）の仮確定。
  - この段階ではまだゲーム内の時間やターンは消費されず、スタミナやMPなどのコストも引かれません。

### 第2段階：予備動作 / 事前判定

- **日本語:** 予備動作 / 発動前チェック

- **英語:** **Anticipation** (アンティシペーション) / **Pre-Action Check** (プレ・アクション・チェック)
- **概要:** 行動を実際に起こす「直前」の準備・チェック状態。
- **ローグライクにおける役割:** * 「恐怖状態で行動できない」「麻痺で動けない」「混乱で行動がランダムな方向に変わる」といった**状態異常による行動制限・変更の割り込み**をここで処理します。
  - 呪文などの「詠唱（チャージ）」が必要な大技の場合、このフェーズで数ターン待機状態になります。

### 第3段階：実行

- **日本語:** 実行 / 発動

- **英語:** **Execution** (エグゼキューション) / **Activation** (アクティベーション)
- **概要:** キャラクターが実際に体を動かし、スキルや攻撃を放った状態。
- **ローグライクにおける役割:** * ここで初めて、ゲーム内のログに「〇〇は攻撃を繰り出した！」等のテキストが表示されます。
  - この瞬間に、攻撃の軌道上にいる対象（エネミーや壁など）と、命中率・回避率の計算（**Check**）が行われます。

### 第4段階：成否結果

- **日本語:** 成否結果 / 判定

- **英語:** **Outcome** (アウトカム) / **Result** (リザルト)
- **概要:** 実行された行動が「成功したか失敗したか」のフラグと、その数値を確定させる状態。
- **ローグライクにおける役割:** * 計算に基づき、以下の状態（ステート）に分岐します。
  - **Success (成功) / Hit (命中):** ダメージ計算や効果（デバフ付与など）を確定。
  - **Failure (失敗) / Miss (回避):** 「攻撃をかわされた！」「呪文がかき消された！」などの失敗結果を確定。
  - **Critical (会心):** 特殊な倍率計算などを適用。

### 第5段階：事後処理 / 解決

- **日本語:** 事後処理 / 解決 / 硬直

- **英語:** **Resolution** (レゾリューション) / **Post-Action Recovery** (ポスト・アクション・リカバリー)
- **概要:** 確定した結果を実際にゲーム世界（HPやログ、アニメーション）に適用し、キャラクターが次の行動を待つ状態（Idle）に戻るフェーズ。
- **ローグライクにおける役割:** * ダメージによるHP減少、死亡判定、ノックバックの処理。
  - 「攻撃成功時にHPを吸収する」「カウンター罠が発動する」などの**事後割り込み**を処理します。
  - このフェーズの終了をもって、そのキャラクターの「1アクション（ターン）」が完了し、エネルギーや行動権（Speed / エネルギーシステム）が消費されます。

---

## 3. ローグライクでの具体例（通常攻撃の場合）

| 段階 | 処理内容 | ログ・ゲーム画面の挙動 |
| :--- | :--- | :--- |
| **1. Intent** | プレイヤーが東向きに攻撃キーを押す | キャラクターの向きが東に変わる（まだ何も起きない） |
| **2. Pre-Action** | 混乱状態のため、攻撃方向が「北」に化ける | ログにはまだ何も出ない |
| **3. Execution** | 北隣にいるスライムに対して攻撃を実行 | 「プレイヤーはスライムを攻撃した！」とログに表示 |
| **4. Outcome** | 命中判定（ダイス）の結果、**Miss（失敗）** と判定 | （内部的にダメージ0が確定） |
| **5. Resolution** | 攻撃アニメーションが終わり、行動権を消費 | 「しかし 攻撃はかわされた！」とログに表示され、スライムのターンへ |

---

## 4. プログラミング（C# / TypeScriptなど）での実装イメージ

システム内でフェーズを管理するための `Enum（列挙型）` の定義例です。

```共通コード
// 行動の段階を管理する列挙型
enum ActionPhase {
    Intent,       // 1. 企図・入力
    PreAction,    // 2. 状態異常チェック・チャージ
    Execution,    // 3. 実行・判定発生
    Outcome,      // 4. 成否の確定 (Success / Failure)
    Resolution    // 5. 結果の適用・ターン終了処理
}

// 成否結果を表現する列挙型
enum ActionResultType {
    Success,
    Failure,
    Critical,
    Evaded,
    Blocked
}
```
